# Automate Intelligence – AI SaaS Landing Page

Адаптивный лендинг для AI-сервиса. Проект выполнен по макету из Figma и демонстрирует навыки современной вёрстки, работу со сборкой «Чертоги Фрилансера 4» и адаптивную кроссбраузерную вёрстку.

👉 **Живая страница:** [automate-ai-landing](https://allanur77.github.io/automate-ai-landing/)

## О проекте

Этот проект — полностью сверстанная и выложенная в интернет посадочная страница (landing page) для гипотетического AI-продукта. Основная цель: показать, как я умею превращать дизайн-макет в работающий, быстрый и адаптивный сайт.

### Что внутри

- **Вёрстка по макету** (Figma) с точностью до отступов и размеров.
- **Адаптив под все устройства:** десктоп, планшет, мобильный телефон. На мобильных устройствах меню превращается в бургер.
- **Современные CSS-эффекты:** градиенты, `backdrop-filter` (эффект стекла), плавные ховеры для кнопок и карточек.
- **Оптимизация производительности:** все изображения конвертируются в WebP, шрифты загружаются асинхронно.
- **Чистая архитектура кода:** методология БЭМ, компонентный подход, разделение стилей и скриптов.

## Технологии и инструменты

- **Сборка:** Vite + «Чертоги Фрилансера 4» — мощный стартовый шаблон для верстальщика.
- **Языки:** HTML5, SCSS (препроцессор CSS), JavaScript (для бургер-меню и аккордеона).
- **Методология:** БЭМ для именования классов.
- **Графика:** SVG (иконочный шрифт и спрайты), WebP для растровых изображений.
- **Деплой:** GitHub Pages для бесплатного хостинга.

## Как я это делал

1.  **Настройка окружения:** Развернул сборку «Чертоги 4», установил зависимости, настроил пути и алиасы.
2.  **Вёрстка компонентов:** Разбил макет на логические блоки (шапка, hero-секция, карточки тарифов, FAQ, футер). Каждый блок вынес в отдельный компонент.
3.  **Адаптив:** Прописал медиа-запросы и использовал миксины сборки для плавного изменения размеров шрифтов (`adaptiveValue`).
4.  **Интерактив:** Настроил бургер-меню, добавил плавные появления блоков (можно было бы добавить AOS, но оставил «нативно»).
5.  **Деплой:** Собрал финальную версию (`npm run build`) и залил на GitHub Pages.
